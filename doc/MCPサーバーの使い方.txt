MCPサーバーの簡単な使い方

概要：
このドキュメントは、ローカルで動作する簡単なMCP(Model Context Protocol)サーバーのセットアップと起動方法を説明します。

前提：
- Node.jsがインストールされていること（推奨：16.x以上）
- このリポジトリをクローン済みで、作業ディレクトリがプロジェクトルートであること
- 依存関係はpackage.jsonに定義されています

1. 依存関係のインストール
プロジェクトルートで以下を実行して依存関係をインストールします（すでに実行済みの場合は省略可）。

npm install

2. 環境変数
このサンプルでは特別な環境変数は不要ですが、外部サービスやAPIキーを使う場合は環境変数を用意してください。環境変数は`.env`やシェルのエクスポートで設定できます。


3. サーバーの起動
通常はpackage.jsonのスクリプトにstartやdevが定義されています。以下のどちらかを実行してサーバーを起動します。

npm run dev    # 開発用（ts-node/watchなど）
npm start      # 本番用ビルドから起動

4. 動作確認
サーバーが起動したら、ブラウザやcurlでルートにアクセスして動作確認を行います（例：http://localhost:3000）。

curl http://localhost:3000/

5. ログとエラー対応
起動時にエラーが発生した場合は、ターミナルのエラーメッセージを確認してください。TypeScriptの型エラーや依存関係不足が多いので、必要に応じて以下を実行します。

npm run build   # TypeScriptをコンパイルする場合

6. 開発のヒント
- 既存の`src/`ディレクトリにあるコードを参照してエンドポイントや設定を確認してください。
- linterやformatter、テストがある場合はそれらを先に通すようにしてください。

7. 参考
- プロジェクトルートのREADMEや`context/TypeScript_SDK_README.md`を参照してください。

以上。問題があれば教えてください。
